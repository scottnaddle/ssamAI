## 2. 평가자료 패턴

### 2-1. 형성평가 (차시별)
- **형식**: HWP 파일, 1~2쪽
- **문항 구성**: 보통 5~10문항 (선택형 + 단답형 + 서술형 혼합)
- **정답 표시**: 마지막 페이지에 정답표 별도 제공
- **난이도**: 하(기초), 중(보통), 상(심화) 표시

### 2-2. 수행평가 (단원별)
- **채점 기준 (3요소)**:
  1. **채점 요소**: 평가 요소 기반, 문항에 맞게 구체화
  2. **채점 요소별 척도**: 숫자(3,2,1) 또는 명칭(상/중/하) 부여
  3. **척도별 수행 특성**: 각 척도에서 보일 일반적 정보 진술
- **예시 (4차시 - 건의문 세부 내용)**:
  | 채점 요소 | 척도 | 수행 특성 |
  |-----------|------|-----------|
  | 문제 상황 정리하기 | 3 / 2 / 1 | 명확+피해사례 → 정리했으나 부족 → 간명하게 못함 |
  | 요구 사항 정리하기 | 3 / 2 / 1 | 명확+구체 → 있으나 불명확 → 관련성 부족 |
  | 예상 독자 분석하기 | 2 / 1 | (생략) |

### 2-3. 자기평가 도구 (경기도교육청)
- **활용 방법**:
  1. 체크리스트 활용 (수업 중 학생 자가 평가)
  2. 루브릭 평가표 작성 (개별 피드백, 재지도)
  3. 학습 일지 작성 (평가 결과 → 학습 목표 재설정)
  4. 동료평가 연계 (협력학습 연결)
- **구성 요소 4가지**: 목표 설정 → 자기 관찰 → 자기 반성 → 피드백

## 3. 행정문서 표준

### 3-1. 공문서 작성 법규
- **행정안전부 '행정업무운영 편람'** + **국어기본법 '어문규범'**
- **날짜 표기**: `2026. 6. 24.(수)` 형식 (요일 괄호 표기)
- **붙임 표기**: '붙임 참조' (O) vs '붙임 참고' (X)
- **금액 표기**: 한글/숫자 원칙 준수
- **물결표(~)** vs **붙임표(-)**: 용도 구분
- **여러 항목 정렬**: 두 줄 이상일 때 정렬 규칙 적용
- **표 본문**: 정확한 그리는 방법 별도 가이드

### 3-2. 교원 행정문서 유형 (수집된 양식)

### 3-3. 공문서 작성 교육과정 (홍익대 평생교육원 사례)
- 교재: "올바른 공문서 작성의 이해" (3H)
- 주요 챕터:
  - 공문서 작성 시 적용 규정 우선순위
  - 항목 두 줄 이상 정렬
  - 절대 틀릴 수 없는 '날짜 표기법'
  - '적극 협조해 주시기 바랍니다' (O) vs 오탈자 사례
  - 금액 표기 원칙
  - '붙임'과 '1부' 정확히 표기
  - 물결표(~)와 붙임표(-) 사용 원칙

## 5. 핵심 시사점 (Phase 2 설계용)

### 5-1. 패턴 3가지
1. **차시별 자료 패키지**: 한 차시 = (수업 PPT + 수업 과정안 + 활동지 + 형성평가) 4종 세트. 이 구조가 표준
2. **공문서 표준 양식 준수**: 행정안전부 편람 + 국어기본법 → 모든 행정문서 생성기는 이 규칙 자동 적용 필요
3. **채점 기준 3요소 (수행평가)**: 채점 요소 + 척도 + 척도별 수행 특성 → 루브릭 생성기 만들 때 필수

### 5-2. 기존 스킬 개선 제안
1. **pptx-generator**: 차시 표기 표준 `[N차시] 제목 (교과서 X~Y쪽)` 패턴 자동 반영
2. **hwpx**: 형성평가 5지선다 + 정답표 자동 생성 패턴 추가
3. **신규 스킬 후보**:
   - `lesson-plan-generator` (수업 과정안 HWP/PDF)
   - `formative-assessment-generator` (형성평가)
   - `rubric-generator` (수행평가 루브릭)
   - `official-letter-generator` (행정공문)
   - `home-letter-generator` (가정통신문/동의서)

### 5-3. 일일 수집 트리거 인프라 (구축 완료)
- 스크립트: `samples/research/scripts/daily-research.sh`
- 설치: `samples/research/scripts/install-launchd.sh install`
- 트리거 시간: 매일 오전 6시
- 등록: Phase 2 진입 시 `./install-launchd.sh install` 실행
