# UAT (User Acceptance Testing) 시나리오

**목적**: 실제 선생님이 사용하기 전, AI가 선생님 페르소나로 시나리오 실행하여 시스템 검증
**대상 시스템**: ssamAI (Phase 1+2+3 완료)
**테스트 일자**: 2026-06-25

---

- **T2 박민호**: "독서 토론 수행평가 채점 기준 만들어야 함. 루브릭 필요."

- **T5 정미경**: "학기 초 교육청 공문 보내야 함. 연수 보고서도."

## 2. 테스트 시나리오 (7개)

### 시나리오 1: 선생님 온보딩 (T1 김선영)
- 1.1: `/signup`에서 이메일 가입
- 1.2: 로그인 → JWT 토큰 저장
- 1.3: `/onboarding` 5단계 페르소나 입력
  - 1단계: 학교급 (초등)
  - 2단계: 학년 (4학년)
  - 3단계: 과목 (담임 - 전과목)
  - 4단계: 수업 스타일 (실험·탐구 중심)
  - 5단계: 교육 철학 (개별 맞춤)
- 1.4: `/chat` 진입 확인
- **성공 기준**: 페르소나 저장 성공 + `/chat` 도달

### 시나리오 2: 단일 스킬 생성 (T3 이수진)
- 2.1: `/library` → `🛠️ 개별 스킬` → `📝 수업 과정안` 선택
- 2.2: 입력 폼 작성 (수학 I, 고등학교 1학년, 1학기, 다항식 단원, 2차시, ...)
- 2.3: 생성 → HWPX 파일 다운로드
- 2.4: 파일 열어서 내용 검증 (도입-전개-정리 구조)
- **성공 기준**: HWPX 파일 정상 생성 + 내용 적합

### 시나리오 3: 오케스트레이터 (T1 김선영)
- 3.1: `/library` → "✨ 한 줄로 요청하세요" → "현장학습 자료 만들어줘" 입력
- 3.2: "🚀 생성" 클릭 → 워크플로 자동 감지 → 다중 스킬 실행
- 3.3: 가정통신문 HWPX 다운로드
- 3.4: 만족도 ⭐ 위젯 표시 확인
- **성공 기준**: 워크플로 감지 정확 + 가정통신문 생성 + ⭐ 위젯

### 시나리오 4: 만족도 평가 (T3 이수진)
- 4.1: 오케스트레이터 결과 후 ⭐ 5점 클릭
- 4.2: "⭐ 평가 완료 — 감사합니다" 토글 확인
- 4.3: `/api/library/feedback`에 기록 확인
- 4.4: 같은 스킬 2회 사용 → 만족도 평균 갱신 확인
- **성공 기준**: 1-5점 저장 + 평균 계산 + UI 토글

### 시나리오 5: 업데이트 후보 확인 (T4 최영식)
- 5.1: `/library` → `🌐 커뮤니티` 탭
- 5.2: "🔧 스킬 자동 업데이트 (Loop 2 완성)" 섹션 확인
- 5.3: 5명 합의 + 평균 3.5+ 패턴 READY 배지 확인
- 5.4: 패턴별 빈도 / 만족도 / suggested_update 표시 확인
- **성공 기준**: READY 패턴이 1개 이상 + suggested_update markdown 표시

### 시나리오 6: auto-merge 미리보기 + 적용 (T5 정미경)
- 6.1: READY 패턴의 "🔧 미리보기" 버튼 클릭
- 6.2: 모달에서 diff 표시 확인
- 6.3: "✅ 적용" 클릭 → 파일 변경 확인
- 6.4: 백업 파일 (.bak.{timestamp}) 생성 확인
- 6.5: 중복 적용 시도 → 409 에러 확인
- **성공 기준**: dry-run diff + confirm 백업 + 409 dedup

### 시나리오 7: 개인화 (모든 선생님)
- 7.1: 각 선생님이 `/library` → `✨ 생성` 탭 진입
- 7.2: "💡 추천" 섹션 표시 확인 (사용 이력 있는 선생님만)
- 7.3: top 3 스킬 버튼 + 다음 추천 workflow 표시
- 7.4: 추천 스킬 클릭 → 오케스트레이터 실행 확인
- 7.5: 시간대 배지 (🌅/☀️/🌙) 표시 확인
- **성공 기준**: 개인화 섹션 + top 3 + 다음 추천 + 시간대

## 3. 성공/실패 기준

### Pass
- 5명 × 7 시나리오 = 35개 체크포인트 중 **80% 이상** 통과
- 핵심 워크플로 (오케스트레이터, 만족도, auto-merge) 100% 통과
- 안전장치 (백업, 409, CORS) 모두 작동

### Fail
- 핵심 워크플로 실패
- 파일 손상 (백업 없음, git 추적 안 됨)
- 보안 블로커 (토큰 노출, CORS 우회)
- 데이터 손실 (Neo4j write 실패)
