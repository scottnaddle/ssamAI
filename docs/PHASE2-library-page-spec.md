## 2. 생성 모달 (Create Modal)

선생님이 [만들기] 클릭 시 5종 스킬 중 하나에 맞는 모달:

### 2-1. 수업 과정안 모달
```yaml
교과명: [드롭다운]
학년: [드롭다운]
학기: [1학기/2학기]
단원명: [입력]
차시: [N/전체] ← 슬라이더
쪽수: [입력]
학습주제: [입력]
활동유형: [체크박스 다중]
학습환경: [체크박스 다중]
준비물: [입력]
학습목표: [동적 추가 1~3개]
[생성하기]
```

### 2-2. 형성평가 모달
```yaml
교과명 / 학년 / 단원명: [공통]
문항 수: [10 ▾]
난이도: [하: 4] [중: 4] [상: 2] (슬라이더)
문항 유형: [5지선다 ☑] [단답형 ☑] [서술형 ☑]
주제 키워드: [입력] (자동 문항 생성)
[생성하기]
```

### 2-3. 루브릭 모달
```yaml
교과명 / 학년 / 단원명: [공통]
평가명: [입력]
평가 방식: [관찰/프로젝트/발표/실험보고서]
채점 요소 수: [3~5]
척도: [3단계/4단계/P-F]
[생성하기]
```

### 2-4. 행정공문 모달
```yaml
문서 종류: [공문/연수보고서/출장신청서/회의록/휴가신청서]
수신/제목/날짜/... (종류별 자동 필드)
[생성하기]
```

### 2-5. 가정통신문 모달
```yaml
문서 종류: [현장학습/안전체험/학교폭력안내/일반]
행사명 / 일시 / 장소 / 비용 / ... (종류별 자동 필드)
[생성하기]
```

## 3. Backend API 엔드포인트

```
POST   /api/library/generate        # 5종 스킬 라우팅
GET    /api/library/my               # 내 자료 목록
GET    /api/library/my/:id           # 내 자료 상세
PATCH  /api/library/my/:id           # 내 자료 수정
DELETE /api/library/my/:id           # 내 자료 삭제
POST   /api/library/my/:id/share     # 커뮤니티 공유
GET    /api/library/community        # 커뮤니티 자료
GET    /api/library/community/:id    # 커뮤니티 자료 상세
POST   /api/library/community/:id/fork  # 내 것으로 가져오기
POST   /api/library/feedback         # 만족도 + 편집 데이터
GET    /api/library/stats            # 사용 통계
```

## 4. 데이터 모델

### 4-1. `templates` 테이블 (PostgreSQL)
```sql
CREATE TABLE templates (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  category VARCHAR(50),  -- lesson-plan/assessment/rubric/official-letter/home-letter
  title TEXT,
  content JSONB,  -- 생성된 HWPX 데이터 (S3 참조)
  parent_id UUID,  -- 파생된 경우 원본 ID
  is_community BOOLEAN DEFAULT FALSE,
  is_anonymous BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ,
  updated_at TIMESTAMPTZ,
  usage_count INT DEFAULT 0,
  satisfaction_avg FLOAT,
  edit_ratio FLOAT  -- 자동생성 대비 수정 비율
);
```

### 4-2. `pattern_frequency` (Neo4j)
```cypher
(:Pattern {name: "학부모 확인란", type: "home-letter"})
  -[:USED_BY {count: 12, last_seen: "2026-06-24"}]->
  (:Template {id: "..."})
```

## 5. 사용자 시나리오 (실제 흐름)

### 시나리오 A: "이번 주 2차시 자료 일괄"
```
1. [생성] 탭 → [이번 주 2차시 자료 전부] 클릭
2. 모달: 교과=과학, 학년=3학년, 단원=식물의세계, 차시=2/5
3. [생성하기] 클릭
4. 4~5초 후 결과:
   - 01-수업과정안_광합성.hwpx (45KB)
   - (02-차시별 PPT는 별도 진행)
   - 03-활동지.hwpx
   - 02-형성평가_광합성.hwpx (56KB)
5. "절약 시간: 4시간" 배지 표시
6. 다운로드 / 수정 / 공유 옵션
```

### 시나리오 B: "선생님 패턴 학습 (Loop 2)"
```
1. 선생님 A가 형성평가 2번 문항을 "해설 추가" 패턴으로 3회 수정
2. teacher-template-loop 자동 감지
3. 다른 선생님 5명도 같은 패턴 추가
4. "🤖 '5지선다 해설 추가' 패턴이 formative-assessment-generator 스킬에 추가될 예정" 알림
5. 승인 → 자동 업데이트
6. 다음 생성부터 모든 선생님에게 자동 적용
```

### 시나리오 C: "개인화 추천 (Loop 3)"
```
1. 선생님이 매주 3학년 과학 형성평가 만듦 (3주 연속)
2. 시스템: "OOO 선생님, 자주 만드시던 3학년 과학 5지선다 10문항 + 정답표 형식으로 생성할까요?"
3. 선생님: "응"
4. 자동 적용 (한 번 더 "응" 누르면 그 이후 기본값)
```

## 6. 메트릭 / KPI

### 6-1. 페이지 레벨
- **탭 전환 비율**: 생성 / 내 자료 / 커뮤니티
- **평균 만족도**: ⭐ 4.2+ 목표
- **공유율**: 생성 자료 중 커뮤니티 공유 비율
- **수정 비율**: 자동 생성 대비 사용자 수정 비율 (낮을수록 좋음)

### 6-2. 시스템 레벨
- **시간 절약**: 선생님 1인당 주 12시간+ 목표
- **재방문율**: 주 3회+ 목표
- **생성 건수**: 주 20건+/선생님 목표
- **5명 합의 패턴**: 주 1+ 목표

## 10. 권한 / 보안

- **내 자료**: 본인만 읽기/수정/삭제 가능
- **커뮤니티 공유**: 작성자가 명시적으로 공유 (기본 OFF)
- **익명 옵션**: 커뮤니티 공유 시 익명/실명 선택 가능
- **학교급 표시**: 커뮤니티 자료에 학교급 (초등/중등)만 표시, 학교명은 비공개 (Phase 2.5)
- **교육청 보고 자료**: 공유 불가 (민감)
